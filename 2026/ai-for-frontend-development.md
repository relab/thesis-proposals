# Bachelor: AI for frontend development

## Administrative

- Supervisor: Leander Jehl
- Group Composition: 1-3 students (the scope can be adjusted based to the number of students)
- Bachelor thesis

## Prerequisites

- Proficiency in Web development (HTML, CSS, JavaScript)
- Willingness to use AI tools (e.g., ChatGPT, GitHub Copilot)

### AI Tools

You are expected to use AI tools in this project.
However, you are responsible for the correctness and quality of the code and the report.
Verify AI-generated code and statements with tests, measurements, and appropriate citations in the report.
Be prepared to explain and justify design decisions and any non-trivial generated code during code reviews.
Inaccuracies, misleading content, or design and stylistic similarities that indicate significant reliance on AI without critical review may negatively affect the grade.
Briefly document significant AI assistance (usage log and citations) to make provenance clear.

## Background and Motivation

AI tools are revolutionizing software development, increasing productivity of both experienced and novice developers.
This thesis aims to explore the state of the art of AI assisted frontend development, focusing both on what tools are available and their applicability in a learning context.

## Project Description

The goal for this project is to investigate AI tools for frontend development.
The goal is to answer the following questions:
- Which tools exist that are specifically tailored for frontend development and what do they offer compared to general purpose tools? 
Any information, on how these tools are used should also be included.
- Which tools exist that are useful for educative purposes and how can they be applied in class room?

### Steps in the project

1. **Exploration**: Explore what AI tools exist and how they are used? Also explore popular opinions on these tools, by researching blog posts and forum discussions.

2. **Try out**: Where possible try different tools and compare their capabilities with standard tools, like GitHub Copilot.

3. **Application for education**: Evaluate how tools can be applied in education and classroom settings. This includes both availability as free tools in the classroom, as well as suitability for supporting learning.

4. **Interview**: Optionally, we will try to get some industry contacts to participate in an interview on how AI tools for frontend development are used in practice.
